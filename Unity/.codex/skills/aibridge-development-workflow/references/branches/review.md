# 审查分支

## 适用场景

用户要求 review、audit、检查风险、设计评审、只读分析，且未明确要求修改文件时，进入审查分支。

## 进入规则

1. 先确认项目偏好中审查分支已启用。
2. 默认只读，不直接修改文件。
3. C# 语义关系查询是否优先使用 Code Index，以 `project-workflow-preferences.md` 和当前 Code Index 可用状态为准。

## 输出规则

- Findings 先行，按严重程度排序。
- 每条 finding 必须绑定具体文件/行号/证据。
- 区分 confirmed、refuted、uncertain。
- 如果没有发现问题，明确说明，并列出剩余测试缺口或残余风险。
- 用户确认修复后，交接到实施分支；如果实施分支被禁用，先请求确认。

## 禁止事项

- 不把猜测写成 confirmed finding。
- 不因审查发现问题而擅自改代码。
- 不展开无关重构建议。
