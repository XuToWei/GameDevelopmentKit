# 多语言

GDK 使用一份 `Design/Excel/Localization.xlsx` 同时生成运行时字典、语言清单、资源路径、编辑器预览数据和强类型 Key。多语言导出是 ExcelExporter 的固定后处理步骤。

## 数据格式

工具会遍历工作簿中的所有 Sheet。只有左上角单元格以 `##` 开头的 Sheet 才会参与导出。

表头必须包含一个名为 `key` 的列，其余有效列名必须与 `GameFramework.Localization.Language` 枚举完全一致，例如：

| `##desc` | `key` | `ChineseSimplified` | `English` |
| --- | --- | --- | --- |
| 菜单开始按钮 | `Menu.StartButton` | 开始 | Start |
| 退出提示 | `AskQuitGame.Message` | 是否确认退出游戏？ | Quit the game? |

首列以 `##` 开头的行会被当作说明行跳过。每个有效 Sheet 都可以提供 Key，但同一语言下的 Key 不能重复。

## 执行导出

先编译 `Kit.sln`，再使用任一入口：

```text
Game/Tool/ExcelExporter
Game/Tool/ExcelExporterForJson
Design/Excel/gen all bin.bat
Design/Excel/gen all json.bat
```

`--Customs=Check` 只校验 Luban，不导出 Localization。二进制与 JSON 模式会生成不同扩展名，资源路径代码也会同步更新。

## 生成产物

### 语言资源

```text
Unity/Assets/Res/Localization/<Language>/Localization.bytes
```

JSON 模式改为 `Localization.json`，并删除同目录旧的 `.bytes`；切回二进制时执行相反清理。

### 公共生成代码

| 文件 | 用途 |
| --- | --- |
| `Game/Generate/Localization/AssetUtility.Localization.cs` | 根据 Language 生成资源路径 |
| `Game/Editor/Generate/Localization/LocalizationReadyLanguage.cs` | 编辑器可选语言列表 |
| `UXTool/.../EditorLocalizationTool.Generate.cs` | UXTool 编辑器预览数据 |

### 业务 Key

| 模块 | 文件 |
| --- | --- |
| ET | `Game/ET/Code/Model/Generate/LocalizationKey.cs` |
| GameHot | `Game/Hot/Code/Generate/LocalizationKey.cs` |

Key 常量默认使用简体中文列生成注释；没有简体中文时使用第一种已导出语言。点、斜杠和空格会被整理成合法字段名，并统一添加 `Key_` 前缀。

末尾带数字的同前缀 Key 还会生成选择方法。例如 `test1`、`test2` 会生成 `LocalizationKey.Get_test(int order)`。

## 启动时加载

`ProcedureLaunch` 根据保存设置或编辑器配置确定当前语言。`ProcedurePreload` 随后执行：

```csharp
await GameEntry.Localization.LoadLanguageAsync(
    GameEntry.Localization.Language);
```

`LoadLanguageAsync` 会清空旧字典，恢复内置字典，再通过 GF Resource 读取当前语言文件。

## 代码使用

优先使用生成常量，避免字符串拼写错误：

```csharp
string text = GameEntry.Localization.GetString(
    LocalizationKey.Key_Menu_StartButton);
```

需要格式化参数时，先取本地化模板，再按项目约定格式化。Key 不存在时应关注 GF 日志，不要在业务层静默回退成错误文案。

## UI 文本

项目的 UXTool 已把 GF Localization 接入 `UXText` 与 `UXTextMeshPro`。不需要额外挂旧版 `Localize` 脚本。

静态文本配置：

1. 使用 `UXText` 或 `UXTextMeshPro`。
2. 关闭 `Ignore Localization`。
3. `Localization Type` 选择 `RuntimeUse`。
4. `Localization ID` 填写 Excel 中的 Key。

动态文本可选择 `Preview`，用 `Preview ID` 在编辑器中预览排版，运行时仍由业务代码设置最终文本。

## 编辑器预览

UXTool 初始化后会读取生成的语言清单与字典。编辑器预览可切换具体语言，也可切换 `ShowKey` 或 `NoWord` 检查 Key 绑定和布局溢出。

预览语言与 GF 当前语言不一致时，运行时辅助器会抛出错误，避免界面显示混合语言。

## 切换语言

当前 Demo 在设置中保存语言名称，并提示重启后生效。下次启动时 `ProcedureLaunch` 设置语言，预加载流程重新读取字典。

若业务需要即时切换，应同时更新 GF 字典和 UX 文本：

```csharp
GameEntry.Localization.Language = language;
await GameEntry.Localization.LoadLanguageAsync(language);
LocalizationHelper.SetLanguage((LocalizationHelper.LanguageType)language);
```

切换期间应阻止重复操作，并确保界面在加载完成后刷新。项目默认流程不负责跨场景并发切换。

## 常见问题

### 某个 Sheet 没有导出

检查左上角是否以 `##` 开头，表头是否包含 `key`，以及首个数据行的首列是否不是说明标记。

### 提示 Language 不存在

列名必须与 `GameFramework.Localization.Language` 枚举名称一致，区分拼写。不要填展示名称或语言代码。

### 提示 Key 重复

Key 在整个工作簿、同一语言范围内必须唯一。即使位于不同 Sheet，也不能重复定义。

### 代码常量没有生成

确认 ET 或 GameHot 工程目录存在并完成导表，再检查对应 `LocalizationKey.cs`。生成文件禁止手工修改。

### UI 显示 `UNFILLED TEXT`

确认组件未忽略本地化、ID 正确、当前语言资源已加载，并检查字典中是否真的包含该 Key。
