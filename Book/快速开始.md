# 快速开始

本篇覆盖首次拉取、工具链编译、Unity 模式选择、编辑器运行和 ET 服务端启动。完成后再进入 UI、Entity、热更新或构建专题。

## 1. 环境要求

| 依赖 | 要求 | 用途 |
| --- | --- | --- |
| Unity | `6000.3.18f1` | 客户端与编辑器工具 |
| .NET SDK | 8.x | 服务端、导表、Proto、文件服务器 |
| MongoDB | 按需安装 | ET Demo 的持久化服务 |
| Odin Inspector | 商业授权 | 编辑器检查器与工具面板 |

Unity 的精确版本记录在 `Unity/ProjectSettings/ProjectVersion.txt`。版本不一致时，先确认 API 与资源序列化兼容性，不要直接升级工程。

## 2. 获取源码

仓库的 Luban 扩展是 Git Submodule。首次拉取或切换分支后执行：

```powershell
git submodule update --init --recursive
```

确认 `Tools/Luban/Luban-Extension` 已有内容，再继续编译。

## 3. 编译工具链

在仓库根目录执行：

```powershell
dotnet build Kit.sln
```

`Kit.sln` 包含服务端项目和共享工具。构建产物进入 `Bin/`，供导表、Proto 生成、文件服务器与 ET 服务端使用。

也可在 Unity 中打开 `Game/Build Tool Editor`，点击 `Build Kit.sln`。首次打开 Unity 前使用命令行构建更容易看到完整错误。

## 4. 打开 Unity

用 Unity Hub 打开 `Unity/`，等待脚本和资源导入完成。项目入口场景是 `Unity/Assets/Launcher.unity`，工具栏上的 `Launcher` 会自动切换到该场景并进入播放模式。

如果工具栏尚未出现，先等待编辑器完成编译，并确认 Console 没有编译错误。

## 5. 选择客户端模式

模式开关位于 `Game/Define Symbol`。ET 与 GameHot 互斥，切换时工具会同步调整配置导出、资源规则、裁剪配置和 HybridCLR 设置。

| 目标 | 操作 |
| --- | --- |
| 纯 GF（GameHot） | 使用 GameHot 时选择 `Add UNITY_GAMEHOT`；只用基础 GF 时移除 `UNITY_ET` 与 `UNITY_GAMEHOT` |
| ET | 选择 `Add UNITY_ET` |
| 启用热更新 | 再选择 `Add UNITY_HOTFIX` |

切换后等待 Unity 重新编译。若菜单显示的是 `Remove ...`，表示对应符号已经启用。

## 6. 运行 ET Demo

1. 选择 `Game/Define Symbol/Add UNITY_ET`。
2. 打开 `ET/Build Tool`，将 `CodeMode` 设为 `ClientServer`。
3. 执行 `Game/Tool/ExcelExporter`，确保配置与当前模块一致。
4. 点击工具栏 `Launcher`。

`ClientServer` 会让 Unity 进程同时包含客户端与服务端代码，适合本地 Demo。只连接独立服务端时使用 `Client`；纯服务端裁剪验证可使用 `Server`。

## 7. 运行 GameHot Demo

1. 选择 `Game/Define Symbol/Add UNITY_GAMEHOT`。
2. 执行 `Game/Tool/ExcelExporter`。
3. 点击工具栏 `Launcher`。

GameHot 以 MonoBehaviour 工作流组织热更业务。入口为 `Game.Hot.Init`，加载后实例化 `Assets/Res/Hot/HotEntry.prefab` 并启动自己的 Procedure。

## 8. 独立启动 ET 服务端

先完成 `dotnet build Kit.sln` 和 ET 配置导出。服务端配置位于 `Config/Luban`，运行目录必须是 `Bin/`，因为 `ConfigReader` 使用相对路径读取 `../Config/Luban`。

```powershell
Push-Location Bin
dotnet App.dll --AppType=Watcher --StartConfig=Localhost --Console=1
Pop-Location
```

Watcher 会读取 `StartMachineConfig` 与 `StartProcessConfig`，为本机启动匹配的 ET 进程。只调试单个进程时可使用 `--AppType=Server --Process=<进程Id>`。

停止 Watcher 前，确认其子进程也已退出。若场景使用数据库，启动前应确保 MongoDB 地址与 `StartZoneConfig` 一致。

## 9. 常用工具入口

| 任务 | 入口 |
| --- | --- |
| 导出二进制配置 | `Game/Tool/ExcelExporter` |
| 导出 JSON 配置 | `Game/Tool/ExcelExporterForJson` |
| 生成协议代码 | `Game/Tool/Proto2CS` |
| ET 模板代码 | `ET/Code Creator` |
| ET DLL | `ET/Compile Dll` |
| GameHot DLL | `GameHot/Compile Dll` |
| 完整 HybridCLR 准备 | `HybridCLR/Do All` |
| 构建资源与安装包 | `Game/Build Tool Editor` |

## 10. 首次运行排错

### 找不到 `Tool.exe` 或 `FileServer.dll`

重新执行 `dotnet build Kit.sln`，并检查 `Bin/`。Unity 菜单只是调用这里的构建产物。

### ET 提示未设置 CodeMode

打开 `ET/Build Tool`，选择 `Client`、`Server` 或 `ClientServer`。该选择对应 `UNITY_ET_CODEMODE_*` 编译符号。

### 配置或生成代码缺失

执行一次 ExcelExporter。模式切换会改变 `Design/Excel/ET/luban.conf` 与 `Design/Excel/GameHot/luban.conf` 的 `active` 状态，生成物应与当前模式一致。

### Unity 编译后仍没有 Launcher

检查 Package Manager 是否成功安装 `me.xw.toolbarextension`，再查看 Console 中最早出现的编译错误。
