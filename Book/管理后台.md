# 管理后台 (Admin Dashboard)

## 概述

基于 Blazor Server + MudBlazor 的游戏管理后台，提供服务器管理、玩家管理、版本管理、配置管理、热更新等功能。

## 前置条件

- .NET 8 SDK
- 编译 `Kit.sln`（Admin 项目包含在内）

## 启动方式

### 方式一：独立启动

```bash
cd Bin
dotnet Admin.dll
```

默认端口 5200，访问 `http://localhost:5200`

### 方式二：通过 Aspire 启动

```bash
cd Tools/Shell
start aspire.bat
```

Admin 会作为 Aspire 管理的服务之一自动启动，可在 Aspire Dashboard 中查看状态。

### 方式三：快捷脚本

```bash
cd Tools/Shell
start admin.bat
```

## 默认账户

首次启动自动创建管理员账户：
- 用户名：`admin`
- 密码：`admin`

**建议首次登录后立即修改密码。**

## 配置说明

配置文件：`Share/Admin/appsettings.json`

```json
{
  "Kestrel": {
    "Endpoints": {
      "Http": { "Url": "http://*:5200" }
    }
  },
  "Jwt": {
    "Key": "修改为你的密钥（至少32字符）",
    "Issuer": "Admin"
  },
  "ServerBinPath": "../Bin",
  "Monitoring": {
    "PollingIntervalSeconds": 5
  }
}
```

| 配置项 | 说明 |
|--------|------|
| `Kestrel:Endpoints:Http:Url` | 监听地址和端口 |
| `Jwt:Key` | JWT 签名密钥，生产环境必须修改 |
| `ServerBinPath` | ET 服务器可执行文件目录 |
| `Monitoring:PollingIntervalSeconds` | 实时监控轮询间隔（秒） |

## 功能页面

| 页面 | 路径 | 功能 |
|------|------|------|
| 仪表盘 | `/` | 服务器状态概览、在线玩家数、快捷操作 |
| 服务器管理 | `/servers` | 启停服务器、批量操作、维护模式 |
| 服务器详情 | `/servers/{id}` | 单个服务器性能指标、Fiber/Scene 列表 |
| Fiber 监控 | `/servers/fibers` | 全局 Fiber 列表，按调度器/场景类型筛选 |
| Scene 监控 | `/servers/scenes` | 全局 Scene 列表，按 Zone 分组 |
| 玩家管理 | `/players` | 搜索玩家、封禁/解封/踢出 |
| 玩家详情 | `/players/{id}` | 玩家基本信息、游戏数据 |
| 版本管理 | `/versions` | 版本记录、激活/停用、按平台环境筛选 |
| 版本历史 | `/versions/history` | 激活历史、快速回滚 |
| 配置管理 | `/config` | 配置键值对、游戏公告 |
| 热更新 | `/hotreload` | 代码/配置热更推送、历史记录 |
| 日志 | `/logs` | 实时日志查看，按级别/服务器筛选 |

## ET 服务器集成

Admin 通过 HTTP 与 ET 服务器进程通信，ET 进程需暴露以下端点：

| 端点 | 方法 | 功能 | 对应 Handler |
|------|------|------|-------------|
| `/admin/fibers` | GET | 获取 Fiber 列表 | `HttpGetAdminFibersHandler` |
| `/admin/scenes` | GET | 获取 Scene 列表 | `HttpGetAdminScenesHandler` |
| `/admin/kick` | POST | 踢出玩家 | `HttpPostAdminKickHandler` |
| `/admin/reload` | POST | 热更新（code/config/all） | `HttpPostAdminReloadHandler` |

ET 进程的 HTTP 端口规则：`20000 + ProcessId`

## 健康检查

Admin 提供 `/healthz` 端点，Aspire 自动监控该端点。

## 数据存储

- **LiteDB**：嵌入式数据库，存储在 `Bin/Data/` 目录
  - `admin.db`：用户、玩家、配置、公告、热更记录
  - `versions.db`：版本记录和激活历史
