# 管理后台 (Admin Dashboard)

## 概述

基于 Blazor Server + MudBlazor 的游戏管理后台，提供服务器管理、玩家管理、版本管理、配置管理、热更新等功能。

## 前置条件

- .NET 8 SDK
- 编译 `DotNet/DotNet.sln`（Admin 代码编译进 `Hotfix.dll`）

## 启动方式

### 方式一：独立启动

```bash
cd Bin
dotnet App.dll --AppType=Admin --Process=100002 --StartConfig=Localhost --Console=1
```

默认端口 5200，访问 `http://localhost:5200`

### 方式二：通过 Aspire 启动

```bash
cd Tools/Shell
start aspire.bat
```

Admin 会作为 Aspire 管理的服务之一自动启动，可在 Aspire Dashboard 中查看状态。

### 方式三：快捷脚本

```bash
cd Tools/Shell
start admin.bat
```

## 默认账户

首次启动自动创建管理员账户：

- 用户名：`admin`
- 密码：`admin123`

生产环境必须在首次启动前通过配置或环境变量 `Admin__BootstrapPassword`
修改初始密码；已有用户数据库不会被启动配置覆盖。

## 配置说明

源配置文件：`DotNet/Hotfix/Server/Admin/appsettings.json`。构建时会复制到
`Bin/appsettings.json`，运行时以 `Bin` 中的文件及环境变量为准。

```json
{
  "Kestrel": {
    "Endpoints": {
      "Http": { "Url": "http://*:5200" }
    }
  },
  "Admin": {
    "BootstrapUsername": "admin",
    "BootstrapPassword": "请在首次启动前修改"
  },
  "Monitoring": {
    "PollingIntervalSeconds": 5
  }
}
```

| 配置项 | 说明 |
|--------|------|
| `Kestrel:Endpoints:Http:Url` | 监听地址和端口 |
| `Admin:BootstrapUsername` | 数据库为空时创建的初始管理员用户名 |
| `Admin:BootstrapPassword` | 数据库为空时创建的初始管理员密码 |
| `Monitoring:PollingIntervalSeconds` | 实时监控轮询间隔（秒） |

## 功能页面

| 页面 | 路径 | 功能 |
|------|------|------|
| 仪表盘 | `/` | 服务器状态概览、在线玩家数、快捷操作 |
| 服务器管理 | `/servers` | 启停服务器、批量操作、维护模式 |
| 服务器详情 | `/servers/{id}` | 单个服务器性能指标、Fiber/Scene 列表 |
| Fiber 监控 | `/servers/fibers` | 全局 Fiber 列表，按调度器/场景类型筛选 |
| Scene 监控 | `/servers/scenes` | 全局 Scene 列表，按 Zone 分组 |
| 玩家管理 | `/players` | 搜索玩家、封禁/解封/踢出 |
| 玩家详情 | `/players/{id}` | 玩家基本信息、游戏数据 |
| 版本管理 | `/versions` | 版本记录、激活/停用、按平台环境筛选 |
| 版本历史 | `/versions/history` | 激活历史、快速回滚 |
| 配置管理 | `/config` | 配置键值对、游戏公告 |
| 热更新 | `/hotreload` | 代码/配置热更推送、历史记录 |
| 日志 | `/logs` | 实时日志查看，按级别/服务器筛选 |

## ET 服务器集成

Admin 通过 ET Actor 消息与服务器和 Agent 进程通信，不需要为每个 ET 进程
额外开放 HTTP 管理端口：

| 消息 | 目标 | 功能 |
|------|------|------|
| `Admin2S_GetFibersRequest` | Server Main Fiber | 获取配置中的 Fiber 拓扑 |
| `Admin2S_GetScenesRequest` | Server Main Fiber | 获取配置中的 Scene 拓扑 |
| `Admin2S_KickPlayerRequest` | Server Main → Gate | 定位并断开玩家会话 |
| `Admin2S_ReloadRequest` | Server Main Fiber | 热更新（code/config/all） |
| `Admin2Agent_*` | Agent | 启停进程、部署文件、分发热更 |

## 健康检查

Admin 提供 `/healthz` 端点，Aspire 自动监控该端点。

## 数据存储

- **LiteDB**：嵌入式数据库，存储在 `Bin/Data/` 目录
  - `admin.db`：用户、玩家、配置、公告、热更记录
  - `versions.db`：版本记录和激活历史
